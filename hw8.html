Understanding XQuery

  1. XQuery is used to search XML documents for elements and atributes.1 It's main function is to allow a programmer to extract specific data. It's benifit is in its high level ease of use and versitility/power in extracting data. XQuery can also be used to transform XML into XHTML.2 It has a broad range of application in web service and for use in summary reports.
  2. When hw8.xq is applied to hw8.xml it will select every book in bookstore where the price is less than 30 and return their titles. Breaking down the following code snippet:
    xquery version "1.0";

    for $b in
      doc("hw8.xml")/bookstore/book

    where
      $b/price < 30

    return $b/title
    • The line "xquery version "1.0";" defines the xquery version being used
    • The line "for $b in" creates a variable "$b"
    • The line "doc("hw8.xml")/bookstore/book" looks in "hw8.xml" -> for each "bookstore" -> for each "book" and sets that array of returned elements into "$b"
    • The line "where $b/price < 30" selects all the books stored in $b where the price attribute is less than 30
    • The line "return $b/title" returns the title attribute from all of the selected books in $b

  3. What is XQIB?

    XQIB stands for XQuery In the Browser.1 XQIB can be used with XQuery to perform client side XQuery operations in a browser just like you would with javascript3 To do so one one only needs to include a the script:3
    "<script type="text/javascript"src="path/to/mxqueryjs/mxqueryjs.nocache.js"></script>"
    This could be very useful if one wanted to build a lightweight web application to run entirely client side, reducing the need for server resources while retaining the power and versitility of using XML with XQuery.
